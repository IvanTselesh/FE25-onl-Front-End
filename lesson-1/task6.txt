Скорее всего, отправлять в Github это задание не нужно, но оставлю здесь ради тренировки, а также, чтобы не потерять:

<aside></aside> - имеет косвенное отношение к содержимому (м.б. рассмотрен отдельно от содержимого). Применяется для боковых панелей, отеделенных от основного текста. Закрывающий тег обязателен.

<article></article> - применяется для статей сайта, комментариев. Является независимым объектом страницы, включает в себя "шапку", содержимое и "подвал" (заголовок, текст, дата, автор и т.д.). Должен содержать тег заголовка <h1></h1>, <h2></h2>, ..., <h6></h6>. Минимальная структура:
  <article>
    <h1>Заголовок</h1>
    <p>Текст статьи</p>
  </article>
Закрывающий тег обязателен.

<audio src=""></audio> - аудиозапись на странице, может иметь текст, отображающийся при неработающем айдиофайле. Файл добавляется через аттрибут src="" (для воспроизведения в разных браузерах можно указать несколько источников). Примерная структура:
  <audio src="<адрес>"></audio>
    <audio>
    <source src="<адрес>">
  </audio>

<body></body> - хранит информацию, выводимую на страницу. Используется для размещения обработчиков событий. Открывающий и закрывающий теги являются признаком хорошего тона.

<button></button> - создает кнопку, обладающую возможностью содержать в себе любые элементы HTML, в том числе и картинки.

<footer></footer> - "подвал" страницы с автором, контактами и т.д. Нельзя поместить один footer в другой footer, address, header. В себе не должен содержать <main></main>. Закрывающий тег обязателен.

<form action=""></form> - создает форму для обмена данных между сервером и пользователем, м.б. любое количество в документе, но обрабатываться будет за раз только одна. Отправляет информацию на сервер либо по клику Submit, либо нажатием Enter. Аттрибут action управляет данными в форме. 

<h1></h1> - заголовок первого уровня (Папочка заголовков), отображается крупным шрифтом жирного начертания в отличие от остальных. Является блочным элементом. Кроме того, перед и после заголовка добавляется пустое место. Заголовки очень "любимы" поисковиками, т.е. повышается ценность текста на странице.

<head></head> - служит для данных, целью которых является помощь браузеру в работе с данными. Внутри находятся метатеги, для хранения информации для поисковиков и браузеров. Содержимое, за исключением тега <title></title>, не отображается на странице. Закрывающий тег необязателен. Может содержать в себе элементы: <base>, <basefont>, <bgsound>, <link>, <meta>, <script>, <style>, <title>.

<header></header> - "шапка" сайта, обычно содержит логотип, разделы поисковую форму, название. <header> нельзя вкладывать внутрь другого <header>, а также внутрь элементов <address> и <footer>. Внутри не должно быть элемента <main>. Закрывающий тег обязателен.

<html></html> - содержит в себе все содержимое документа, в том числе <head></head> и <body></body>, страницы, кроме <!Doctype>, закрывающий тег необязателен, но наличие является признаком хорошего тона.

<img src="" alt=""> - отображает картинку на странице в формате GIF, JPEG, SVG или PNG, аттрибут src задает путь к файлу, а, закинув картинку в тег <a></a>, её можно сделать ссылкой. Закрывающий тег не требуется.

<input type="text"> - обеспечивает взаимодействие с пользователем через разные элементы формы: текстовое поле (text), поле с паролем (password), переключатель (radio), флажок (checkbox), скрытое поле (hidden), кнопка (button), кнопка для отправки формы (submit), кнопка для очистки формы (reset), поле для отправки файла (file), кнопка с изображением (image) и др. Закрывающий тег не требуется.

<label for=""></label> - устанавливает связь между определённой меткой, в качестве которой обычно выступает текст, и элементом формы (<input>, <select>, <textarea>). Такая связь необходима, чтобы изменять значения элементов формы при щелчке курсором мыши на текст. Существует два способа связывания объекта и метки. Первый заключается в использовании идентификатора id внутри элемента формы и указании его имени в качестве атрибута for элемента <label>. При втором способе элемент формы помещается внутрь контейнера <label>. Структура: 
  <input id="<идентификатор>"><label for="<идентификатор>">Текст</label>
  <label><input> Текст</label>

<li></li> - определяет отдельный пункт списка. Внешний элемент <ul> или <ol> устанавливает тип списка — маркированный или нумерованный. Закрывающий тег не обязателен.

<link rel="stylesheet" href=""> - связывает с внешним документом, но не создает ссылку в отличие от <a></a>, обычно располагается в <head></head>

<main></main> - включает в себя содержимое, которое является уникальным для данного документа, и не должен включать повторяющиеся разделы сайта, такие как навигация, название сайта, логотип, поисковая форма, баннеры и др. Не должен располагаться внутри элементов <article>, <aside>, <footer>, <header>, <nav> или <section>. Закрывающий тег обязателен.

<menu></menu> - предназначен для отображения списка пунктов меню. Аналогично элементам <ol> и <ul> внутри контейнера <menu> список формируется с помощью <li>.

<meta> - определяет данные (они называются ещё метатеги), которые используются для хранения информации, предназначенной для браузеров и поисковых систем. Разрешается использовать более чем один метатег, все они размещаются в контейнере <head>. Как правило, атрибуты любого метатега сводятся к парам «имя=значение», имена которых определяются ключевыми словами content, name или http-equiv.

<nav></nav> - задаёт навигацию по сайту. Если на странице несколько блоков ссылок, то в <nav> обычно помещают приоритетные ссылки. Также допустимо использовать несколько <nav> в документе. Запрещается вкладывать <nav> внутрь <address>. Закрывающий тег обязателен.

<p></p> - является блочным, всегда начинается с новой строки, абзацы текста идущие друг за другом разделяются между собой отбивкой. Если закрывающего тега нет, считается, что конец абзаца совпадает с началом следующего абзаца или другого блочного элемента.

<script></script> - предназначен для описания скриптов, может содержать ссылку на программу или её текст. <script> может располагаться в заголовке или теле HTML-документа в неограниченном количестве. В большинстве случаев местоположение скрипта никак не сказывается на работу программы. Однако скрипты, которые должны выполняться в первую очередь, обычно помещают в заголовок документа. Имеет аттрибут src, если скрипт внешний.

<select name="" id=""></select> - создать элемент интерфейса в виде раскрывающегося списка, а также список с одним или множественным выбором.Каждый пункт создаётся с помощью элемента <option>, который должен быть вложен в контейнер <select>. Если планируется отправлять данные списка на сервер, то требуется поместить <select> внутрь формы. Это также необходимо, когда к данным списка идёт обращение через скрипты. Структура: 
  <select>
    <option>Пункт 1</option>
    <option>Пункт 2</option>
  </select>

<section></section> - Задаёт раздел документа, может применяться для блока новостей, контактной информации, глав текста, вкладок в диалоговом окне и др. Раздел обычно содержит заголовок. Допускается вкладывать один элемент внутрь другого. Закрывающий тег обязателен.

<span></span> - предназначен для выделения отдельных строк, символов или других строчных элементов для дальнейшего изменения их оформления с помощью стилей. Закрывающий тег обязателен.

<textarea name="" id="" cols="30" rows="10"></textarea> - представляет собой элемент формы для создания области, в которую можно вводить несколько строк текста. В отличие от элемента <input> в текстовом поле допустимо делать переносы строк, они сохраняются при отправке данных на сервер.

Между тегами <textarea> и </textarea> можно поместить любой текст, который будет отображаться внутри поля. Закрывающий тег обязателен.

<title></title> - не является частью документа и не показывается напрямую на веб-странице. Текст заголовка отображается, как правило, на вкладке браузера. Допускается использовать только один <title> на документ и размещать его в контейнере <head>. Закрывающий тег обязателен.

<ul></ul> - устанавливает маркированный список. Каждый пункт списка должен начинаться с элемента <li>. Закрывающий тег обязателен.

<video src=""></video> - видеозапись на странице, может иметь текст, отображающийся при неработающем видеофайле. Файл добавляется через аттрибут src="" (для воспроизведения в разных браузерах можно указать несколько источников). Примерная структура:
  <video src="<адрес>"></video>
    <video>
    <source src="<адрес>">
  </video>
